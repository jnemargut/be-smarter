Decision 1 of 6 · The Method

What is the thinking protocol, exactly?

Strip away decision-kit's domain skills and HTML pages, and what's left is a thinking method. This decision defines it precisely — the phases the model walks through on any task. Everything else (depth, HITL, artifacts, portability) hangs off this.

Research Context
  • Divergent→convergent is a studied, effective strategy — not just a vibe. Decomposing into "a divergent phase that explores the space freely, then a convergent phase that refines selections" measurably improves GPT-4 problem-solving (self-reflection −9.7% infeasible solutions; divergent-convergent +6.5% efficient solutions). arXiv 2512.23601, MacGyver benchmark
  • Multi-agent debate is the strongest divergence mechanism on record — independent perspectives that argue break the "degeneration of thought" failure where a model anchors on its first idea and merely defends it. Cost is proportional to the number of debaters. arXiv 2305.19118 (MAD)
  • More thinking is not monotonically better. Accuracy follows an inverted-U with reasoning length; long chains accumulate errors and models abandon correct answers ("overthinking"). The protocol needs convergence discipline as much as divergence breadth. arXiv 2502.07266, 2410.21333, OptimalThinkingBench
  • Brute-force divergence is losing its edge. Self-consistency (sample 10 chains, vote) gains only 0.4–1.6% on frontier models while cost scales linearly. Structured divergence — different framings, different perspectives — beats more samples of the same thought. arXiv 2511.00751
  • Practitioners value the framing phase most. The most-loved part of Superpowers' brainstorming skill is that it "pressure-tests assumptions using first principles" and challenges the problem statement before any solution work. Solving the wrong problem well is the failure no amount of solution-space thinking fixes. Superpowers plugin, codeminer42 field report
  • Decision-kit's own evidence: auto-picked runs still produce better output — because generating 4 real options with tradeoffs forces structured evaluation even when the human rubber-stamps the recommendation. The decision-with-options is a proven unit of thought.
Options
A

Latent-decision engine

Decision-kit distilled, framing skipped: every task contains 3–7 latent decisions. Find them, generate 4 genuinely different options each, weigh tradeoffs, pick with stated reasoning, execute.

1Excavate the latent decisions in the task
24 options per decision diverge
3Tradeoff table per decision
4Pick + reasoning converge
5Execute on the picks

Purest port of what decision-kit already proved works

Naturally headless — picks + reasoning need no human

Accepts the task as framed — the wrong-problem failure mode survives

Decision framing is awkward for some tasks (debugging, summarizing)

★ Recommended
B

Double diamond — frame, then solve

Two diverge→converge passes. Diamond 1 (problem space): question the framing — what's actually being asked, what would an expert ask, which assumptions are load-bearing — and converge on the real problem + success criteria. Diamond 2 (solution space): run A's decision-option-tradeoff mechanics, converge, execute.

1Framings, expert questions, assumptions diverge
2The real problem + success criteria converge
3Latent decisions → options + tradeoffs diverge
4Picks + reasoning, checked against criteria converge
5Execute, verify against the criteria from step 2

Catches the wrong-problem failure A can't — the phase practitioners value most

Subsumes A: its mechanics power the second diamond

Step-2 criteria give convergence a measurable target (overthinking guard)

Two diamonds on a trivial task is overkill — leans hard on Decision 2 (depth calibration)

C

Adversarial panel

Divergence through disagreement: spin up 3+ perspectives (skeptic, domain expert, user advocate — or task-appropriate personas) that independently attack the task, debate, and a synthesis pass converges on what survives.

1Cast 3+ perspectives for this task
2Independent takes diverge
3Debate round — attack each other's takes
4Synthesis of what survived converge

Strongest anti-anchoring mechanism in the literature (MAD)

Maps naturally onto subagents when the host supports them

Heaviest token cost; diminishing returns on frontier models for routine tasks

Needs host features (or verbose role-play) — hurts the portability goal

D

Question-first scaffold

The lightest protocol: generate the questions an expert would ask before answering, answer them honestly (including "I don't know"), keep an assumptions ledger, then one converge pass produces the answer.

1What would an expert ask first? diverge
2Answer each; log assumptions + unknowns
3Answer the task, citing the ledger converge

Cheapest; near-zero overthinking risk; trivially portable

Self-ask is a proven standalone technique

No structured option generation — tradeoff thinking (the kit's core move) is optional, not forced

Closest to what good models already do; least differentiation

Recommendation: B. It keeps decision-kit's proven decision-option-tradeoff engine but wraps it in the framing pass that catches the one failure that engine can't — solving the wrong problem. C's debate move can live inside diamond 2 as an optional escalation for genuinely hard calls; D is what the protocol collapses to at minimum depth (Decision 2's territory).
