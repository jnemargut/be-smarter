# Example: be-smarter designing itself

This folder is the actual decision record from designing this project, produced by running decision-kit's `/strategize` on the idea before any code or prose existed.

Seven decisions, each researched with four options and a recommendation: the core protocol (double diamond won), depth calibration (scored triage won), the human dial (three positions won), artifacts (machine trace with lazy page rendering won), portability (the standard-conformant skill won), the repo's home (this sibling repo won), and the pitch at the top of the README.

Two things worth noticing as you browse:

1. The chosen design is itself an argument for the method: divergent options plus explicit tradeoffs surfaced choices (lazy page rendering, the reserved `--human=auto` position) that a first-idea answer would have missed.
2. These HTML pages exist because a human was in the loop for this run. A headless be-smarter run on the same question would have produced the trace and the summary, and rendered pages only if someone asked.

Start with [strategy-brief.md](strategy-brief.md), or open [index.html](index.html) for the hub.

(These artifacts predate this repo's style rules and keep their original decision-kit formatting.)
