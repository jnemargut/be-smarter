Decision 5 of 6 · Reach

What does "improves any model's thinking" mean concretely?

"Any model" could mean any coding agent, any chat window, or any product built on an API. Each has a different delivery vehicle. The good news from research: the delivery problem mostly got solved while you weren't looking.

Builds on Decisions 1–4: the protocol is pure text; the trace needs a filesystem; the dial needs an argument convention. Portability decides which of those are hard requirements and which degrade gracefully.
Research Context
  • SKILL.md became an open standard — and it won fast. Anthropic released the Agent Skills spec in December 2025; within 48 hours Microsoft (VS Code) and OpenAI (ChatGPT, Codex CLI) adopted it. By March 2026, 32 tools — Gemini CLI, Cursor, JetBrains Junie, AWS Kiro, Block's Goose, Cline, Windsurf, OpenCode — read the same SKILL.md files. One conformant skill is now inherently cross-tool. paperclipped.de interoperability guide, agensi.io, developers.openai.com/codex/skills
  • The standard has neutral governance. The Agentic AI Foundation (Linux Foundation) stewards AGENTS.md and the skills ecosystem, with 146 member orgs by Feb 2026 — betting on the standard isn't betting on Anthropic. agents.md, AAIF
  • "Copied into Codex's skills directory and it works" is the documented bar — but only if the skill body avoids host-specific tool references and feature assumptions. Host-specific names (Task tool, AskUserQuestion) are where portability dies in practice. interoperability guide, Shibui field notes
  • Superpowers proved the model AND its limit. Its skills run across six hosts from one directory — but its richest behaviors (subagent dispatch) silently degrade on hosts without those features. The lesson: write the core protocol against the lowest common denominator (text in, text out), feature-detect the rest. blog.marcnuri.com, Superpowers repo
  • Your own components sort cleanly by dependency: the double diamond + triage + dial argument are pure text (work anywhere, even pasted into ChatGPT web); the trace file needs a filesystem (32-tool club); page rendering needs file + open (coding agents only). Decision 4 already designed the degrade path (trace → response appendix).

Claude Code first, port later

Write it as a Claude Code skill like decision-kit's others; worry about other hosts when someone asks.

★ Recommended
B

Standard-conformant, host-neutral skill

One SKILL.md written to the open Agent Skills spec: no host-specific tool names, feature-detection for filesystem/subagents/pages with the Decision-4 degrade chain. The README documents install for the major hosts (one copy command each) and ships PROTOCOL.md — a condensed paste-anywhere version for chat surfaces.

be-smarter/ ├── SKILL.md ← spec-conformant, host-neutral ├── PROTOCOL.md ← paste into any chat └── references/ (triage rubric, trace schema, dial spec) README: install = cp -r into any of 32 hosts' skills dir

32 tools read it day one — "any model" is literally true for agents

PROTOCOL.md covers the no-agent case (ChatGPT web, Gemini app)

Feature-detect + degrade is already designed (Decision 4)

Host-neutral wording is a discipline — needs a conformance check in CONTRIBUTING

Can't lean on Claude-only niceties (subagents, AskUserQuestion) in the core path

C

Protocol document first

The repo's core artifact is PROTOCOL.md — a self-contained thinking method anyone pastes anywhere. Skill wrappers are secondary packaging in an adapters/ folder.

be-smarter/ ├── PROTOCOL.md ← the product └── adapters/ ├── skill/SKILL.md └── system-prompt.txt

Maximum conceptual reach — works in literally any chat box

Positions the method, not the packaging, as the contribution

No install story = no habitual use; documents get read once, skills get used daily

Trace/dial/triage all need an agent host anyway — the doc undersells the design

D

Builder layer: system prompt + API recipe

Target people embedding the method in products: a system-prompt block, an API cookbook, maybe a tiny wrapper lib. Coding-agent skill ships as one consumer of it.

be-smarter/ ├── system-prompt.md ├── cookbook/ (OpenAI + Anthropic examples) └── skill/ (one consumer)

Reaches models with no agent host at all

Different audience (product builders), different repo DNA than decision-kit

Doubles the surface to document and maintain for an unproven audience
